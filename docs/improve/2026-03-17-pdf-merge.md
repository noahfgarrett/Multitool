# Feature Improvement Report: PDF Merge

**Date:** 2026-03-17
**Scope:** PDF Merge (Merge mode + Grid Stitch mode)
**Competitors analyzed:** Adobe Acrobat Pro, Bluebeam Revu, Smallpdf/iLovePDF, PDF-XChange Editor

## Executive Summary

12 findings: 5 gaps, 4 refinements, 3 UX improvements
Top recommendation: Page rotation in merge mode — every competitor has it, pdf-lib already supports it, and construction users frequently need to rotate portrait/landscape pages.

## Current Capabilities

**Merge Mode:** Upload, reorder (DnD + buttons), expand pages, include/exclude pages, copy/paste pages, lazy thumbnails, memory meter, merge & download

**Grid Stitch:** Grid setup (1-15×15, presets), per-cell zoom/pan/offset, snap-to-position, auto-align (NCC), region focus, labels, undo/redo, export with gridlines/labels/compression

## Findings (sorted by ROI)

### 1. [GAP] Page Rotation in Merge Mode
**Impact:** Critical | **Effort:** Small | **ROI: ★★★★★**

**What:** Rotate individual pages 90°/180°/270° before merging — essential when combining portrait and landscape documents.

**Why:** Adobe, Bluebeam, PDF-XChange, and even Smallpdf all support page rotation during merge. Construction documents frequently mix landscape plans with portrait specs. Without rotation, users must pre-process pages in another tool.

**Competitors:** All competitors have this as a basic feature.

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (add rotation state + UI), `pdf.ts` (apply rotation in mergePDFs)
- **Approach:** Add a `rotation` field to `PageEntry` (0/90/180/270). Show rotation buttons (↻/↺) on each page thumbnail. In `mergePDFs`, call `page.setRotation(degrees(rotation))` via pdf-lib after copying.
- **Key details:** pdf-lib's `PDFPage.setRotation(degrees(90))` handles this natively. The thumbnail preview should also rotate via CSS `transform: rotate(Ndeg)`.
- **Complexity:** ~40 lines. State + 2 buttons per page + 1 line in merge function.
- **Watch out for:** Rotation affects page dimensions — a portrait page rotated 90° becomes landscape. The thumbnail display should reflect this.

### 2. [GAP] Output File Size Preview
**Impact:** High | **Effort:** Small | **ROI: ★★★★★**

**What:** Show estimated output file size before merging, so users know if the result will be too large for email (~25MB limit) or file shares.

**Why:** Construction plan sets can be 50-500MB. Users need to know the output size before committing to a merge that takes 30+ seconds. Currently there's no size indication until after download.

**Competitors:** Adobe shows output size estimate. Smallpdf shows it prominently.

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (add size estimate display)
- **Approach:** Sum the file sizes of all included files/pages. Display near the "Merge & Download" button: "Estimated: ~42.3 MB". For page-level selection, estimate proportionally: `fileSize * (includedPages / totalPages)`.
- **Key details:** `File.size` gives bytes. Use existing `formatFileSize()` utility. The estimate is approximate (merged PDFs share fonts/resources so actual may be smaller).
- **Complexity:** ~15 lines. Pure computation + display.
- **Watch out for:** Don't over-promise accuracy — label it "Estimated" or "~".

### 3. [REFINEMENT] Smart Output Filename
**Impact:** High | **Effort:** Small | **ROI: ★★★★☆**

**What:** Generate a meaningful default filename instead of "merged.pdf" — use the first file's name or combine file names.

**Why:** Users rename "merged.pdf" every time. A smart default (e.g., "ProjectAlpha_combined.pdf" or "plans_1-3.pdf") saves a step.

**Competitors:** Adobe uses "Binder1.pdf" (unhelpful). Bluebeam uses the first file's name. Smallpdf generates from input names.

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (line 503, `suggestedName` parameter)
- **Approach:** If 1 file: use its name + "_combined". If 2-3 files: join base names. If 4+: use first file name + `_and_${n-1}_more`.
- **Key details:** Strip ".pdf" from input names, rejoin with "_", add ".pdf" suffix.
- **Complexity:** ~10 lines. String manipulation only.
- **Watch out for:** Very long filenames — truncate to 80 characters.

### 4. [GAP] Drag & Drop File Reordering (Visual)
**Impact:** High | **Effort:** Small | **ROI: ★★★★☆**

**What:** Improve the existing drag reorder to show a clear visual indicator of where the file will be dropped (insertion line between files).

**Why:** The current DnD uses HTML5 drag events which don't show a clear insertion point. dnd-kit (already installed for page-level DnD) provides smooth animated reordering with clear drop indicators. Smallpdf's drag UX is frequently praised.

**Competitors:** Smallpdf (smooth animated drag), Adobe (drag with insertion line), iLovePDF (drag between slots).

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (replace HTML5 drag with dnd-kit for file-level reorder)
- **Approach:** The page-level reorder already uses dnd-kit. Extend it to file-level: wrap file rows in `SortableContext` with `verticalListSortingStrategy`. Show a blue insertion line at the drop position.
- **Key details:** dnd-kit is already a dependency. Use `useSortable` hook on each file row. The `DragOverlay` component can show the file being dragged.
- **Complexity:** ~60 lines. Swap HTML5 drag with dnd-kit pattern already used for pages.
- **Watch out for:** Don't break the existing page-level DnD inside expanded files.

### 5. [UX] Merge Preview — See Result Before Downloading
**Impact:** Medium | **Effort:** Medium | **ROI: ★★★☆☆**

**What:** Show a quick preview of the merged output (page thumbnails in final order) before committing to download.

**Why:** Users want to verify page order, rotations, and exclusions are correct before generating a large file. A preview step catches mistakes early.

**Competitors:** Adobe shows merged preview. Bluebeam shows the final set in the Sets panel.

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (add preview mode)
- **Approach:** When user clicks "Merge Preview", generate a flat list of all included pages in merge order. Render their thumbnails in a continuous scroll. The user can review and then click "Download" to commit.
- **Key details:** Reuse the existing lazy thumbnail loading. The preview is the same page list that merge uses, just displayed differently.
- **Complexity:** ~100 lines. New view mode within the merge tool.
- **Watch out for:** Don't re-render all pages at once — use virtual scrolling or lazy loading.

### 6. [GAP] Bookmarks / Table of Contents
**Impact:** Medium | **Effort:** Medium | **ROI: ★★★☆☆**

**What:** Auto-generate bookmarks (PDF outline) in the merged output — one bookmark per input file, labeled with the filename.

**Why:** Merged plan sets with 50+ pages are unusable without bookmarks. Bluebeam auto-generates bookmarks for combined sets. Adobe supports manual bookmark creation.

**Competitors:** Bluebeam (auto bookmarks from filenames), Adobe (manual bookmarks), PDF-XChange (auto TOC).

**Implementation spec:**
- **Files:** `pdf.ts` (`mergePDFs` function)
- **Approach:** After adding all pages from a file, create a bookmark using pdf-lib's `pdfDoc.addBookmark()` pointing to the first page of that file. Label with the filename (minus extension).
- **Key details:** pdf-lib doesn't have a built-in `addBookmark()`, but you can manipulate the document outline via `pdfDoc.catalog.lookup(PDFName.of('Outlines'))`. Alternatively, use the `@nicolo-ribaudo/pdf-lib-bookmarks` npm helper.
- **Complexity:** ~50 lines. Moderate complexity due to PDF outline structure.
- **Watch out for:** PDF outline format is complex — test with various PDF viewers.

### 7. [REFINEMENT] Grid Stitch — Overlap Blending Mode
**Impact:** Medium | **Effort:** Medium | **ROI: ★★★☆☆**

**What:** When tiles overlap in Grid Stitch, blend the overlapping regions instead of hard-clipping. Show a gradient blend at the seam.

**Why:** Construction plan tiles often have overlapping borders. A hard clip line creates visible seams in the stitched output. Blending the overlap produces a seamless result, which is what Bluebeam's Batch Overlay achieves.

**Competitors:** Bluebeam (overlay blending), specialized panorama stitching tools.

**Implementation spec:**
- **Files:** `GridStitchMode.tsx` (export function)
- **Approach:** In the export loop, when two adjacent cells have overlapping content (offset causes overlap), render both images and apply a linear alpha gradient at the overlap boundary. Use pdf-lib's opacity support or render to canvas with gradient masking.
- **Key details:** The auto-align already computes offsets between cells. Use these offsets to determine overlap width. Apply a canvas gradient mask during export compositing.
- **Complexity:** ~80 lines in export function.
- **Watch out for:** Only applicable when cells have non-zero offsets (after auto-align).

### 8. [UX] Keyboard Shortcuts for Page Management
**Impact:** Medium | **Effort:** Small | **ROI: ★★★☆☆**

**What:** Add keyboard shortcuts for common operations in merge mode: Delete to remove selected page, Ctrl+A to select all, arrow keys to navigate between files.

**Why:** Power users (construction PMs managing 20+ files) work faster with keyboard shortcuts. Currently all operations require mouse clicks.

**Competitors:** Adobe (full keyboard support), Bluebeam (keyboard navigation in Sets panel).

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (add keydown handler)
- **Approach:** Add a `keydown` event listener: Delete removes selected file, Arrow Up/Down moves file position, Ctrl+A selects all pages in expanded file. Track `selectedFileIdx` state for keyboard navigation.
- **Key details:** Only active when merge mode is focused (not Grid Stitch). Use same event handler pattern as PDF Annotate.
- **Complexity:** ~40 lines.
- **Watch out for:** Don't conflict with browser shortcuts (Ctrl+A select-all on page).

### 9. [GAP] Password-Protected PDF Support
**Impact:** Medium | **Effort:** Medium | **ROI: ★★☆☆☆**

**What:** Handle password-protected PDFs — prompt for password on upload, decrypt before merging.

**Why:** Corporate documents are often password-protected. Currently the tool silently fails or shows a cryptic error. Adobe and Bluebeam handle this seamlessly.

**Competitors:** All major tools support this.

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (add password prompt), `pdf.ts` (pass password to `PDFDocument.load`)
- **Approach:** Wrap `PDFDocument.load()` in a try/catch. If it throws a password error, show a modal prompting for the password. Retry with `PDFDocument.load(bytes, { password })`.
- **Key details:** pdf-lib supports `PDFDocument.load(bytes, { password: 'secret' })`.
- **Complexity:** ~60 lines. Password modal + retry logic.
- **Watch out for:** Multiple password-protected files should queue prompts, not overlap.

### 10. [REFINEMENT] Grid Stitch — Save/Load Grid Configuration
**Impact:** Medium | **Effort:** Medium | **ROI: ★★☆☆☆**

**What:** Save the current Grid Stitch configuration (grid dimensions, cell assignments, zoom/offset settings) to a file, and load it back later.

**Why:** Construction projects involve repeated stitching of updated plan tiles. Being able to save the grid layout and re-apply it when new tiles arrive saves significant setup time.

**Competitors:** Bluebeam (Sets panel persists across sessions).

**Implementation spec:**
- **Files:** `GridStitchMode.tsx` (save/load functions + UI buttons)
- **Approach:** Serialize grid state (rows, cols, cells with filenames and zoom/offset) to JSON. Save via File System Access API or download. On load, restore grid dimensions and cell assignments, prompting user to re-upload the source files.
- **Key details:** Don't save file contents (too large). Save filenames + cell assignments. On load, show a file picker to re-associate files.
- **Complexity:** ~100 lines.
- **Watch out for:** Files may have been renamed or moved since save. Show clear error messages.

### 11. [UX] Page Count Badge on Each File
**Impact:** Low | **Effort:** Small | **ROI: ★★★☆☆**

**What:** Show the total selected/total pages badge prominently on each file row (e.g., "3/5 pages" when 2 are excluded).

**Why:** When working with page exclusions, users need to see at a glance how many pages are included from each file. The current UI requires expanding the file to see excluded pages.

**Competitors:** Adobe (shows page count per file), Bluebeam (page count in Sets panel).

**Implementation spec:**
- **Files:** `PdfMergeTool.tsx` (file row display)
- **Approach:** In the file row, display `{includedCount}/{totalPages} pages` or `{totalPages} pages` (if no exclusions). Use orange text when some pages are excluded.
- **Key details:** Already have `pages` array on each file. Filter by `!excluded` for included count.
- **Complexity:** ~10 lines.
- **Watch out for:** Don't show exclusion count before pages are expanded (no page data until first expand).

### 12. [REFINEMENT] Grid Stitch — Multi-Page PDF Support
**Impact:** Medium | **Effort:** Medium | **ROI: ★★☆☆☆**

**What:** When uploading a multi-page PDF to Grid Stitch, allow selecting which page to use in each cell (currently only uses page 1).

**Why:** Some construction plan sets deliver multiple sheets in a single PDF. Users need to place specific pages into specific grid cells.

**Competitors:** Bluebeam (full multi-page support in Sets), Adobe (page-level selection).

**Implementation spec:**
- **Files:** `GridStitchMode.tsx` (add page selector to cell), `processFile()` function
- **Approach:** When a multi-page PDF is dropped into a cell, show a page number input or small dropdown. Re-render the selected page's thumbnail.
- **Key details:** Currently `processFile()` renders page 1. Add a `pageNumber` field to `GridCellData` and use it in rendering.
- **Complexity:** ~60 lines.
- **Watch out for:** Re-rendering thumbnails on page change needs to be async.
