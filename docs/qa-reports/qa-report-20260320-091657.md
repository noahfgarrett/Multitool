## Known Bug: Export Fidelity Mismatch

The user reported that exported watermarks don't match the screen preview. The Phase 1 export fidelity tests verified:

- Exported PDF is larger than original (watermark content added) — **PASS**
- Exported page count matches original — **PASS**
- Export filename follows `baseName-watermarked.pdf` pattern — **PASS**
- Canvas preview shows watermark centered at canvas center — **PASS**
- Canvas preview shows watermark in correct quadrant per position — **PASS**
- Tile mode covers 3+ quadrants — **PASS**

**Observation:** The current tests verify the watermark *exists* in the export and the canvas preview is correctly positioned, but they do **not** compare pixel-level positions between the canvas preview and the PDF export. The mismatch bug may be in:
1. The coordinate system translation between canvas (origin top-left) and PDF (origin bottom-left)
2. The `measureFn` approximation in `handleApply` (`t.length * fontSize * 0.5`) vs the actual canvas `ctx.measureText()` used in the preview
3. The font rendering difference between canvas `sans-serif` and pdf-lib's `StandardFonts`

**Recommendation:** The `measureFn` in `handleApply` (line 347) uses `t.length * fontSize * 0.5` as a rough text width approximation, while the canvas preview uses `ctx.measureText()` which gives exact pixel widths. This is the most likely source of positional mismatch. A follow-up fix should either:
- Use a more accurate character-width table for pdf-lib text measurement
- Or render the watermark as an image (canvas → rasterize → embed) to guarantee pixel-perfect fidelity

## Recommendations

1. **Export fidelity mismatch (user-reported bug):** The `measureFn` text width approximation (`length * fontSize * 0.5`) in `handleApply` differs significantly from the canvas `ctx.measureText()`. This causes watermark text to be positioned differently in the exported PDF vs the canvas preview. Consider using pdf-lib's actual text measurement or rasterizing the watermark overlay as an image for pixel-perfect export fidelity.

2. **No undo/redo:** The watermark tool has no undo/redo functionality. If a user accidentally changes settings, they must manually restore them. Consider adding undo for watermark configuration changes.

3. **No preview of exported PDF:** The user cannot see what the exported PDF will look like before downloading. Consider adding an "Export Preview" step.

4. **Session persistence:** Watermark settings are not saved to localStorage. If the user navigates away and comes back, all settings reset to defaults. Consider persisting the last-used watermark configuration.
